# 人机协同交易系统 BRD

**版本：** v1.0  
**状态：** 业务对齐版  
**日期：** 2026-04-05

## 1. 业务定位

本产品是一个面向个人交易者的人机协同系统，包含两个平行子系统：

1. 盲训子系统（提升主观决策能力）
2. 模型子系统（产出可用买卖点提醒模型）

两个子系统业务上相对独立，可共享基础能力，但不互相依赖上线。

## 2. 业务问题

当前交易实践存在两类核心痛点：

1. 决策能力痛点：主观判断容易受情绪和 hindsight 偏差影响，复盘难以结构化。
2. 信号效率痛点：全市场机会密度高，人工筛选效率低，难以稳定识别顶底买卖点。

## 3. 业务目标

### 3.1 盲训子系统目标

1. 建立标准化训练流程，持续提升盘感与决策一致性。
2. 降低错误决策频次，提高策略执行纪律。
3. 形成可复盘的行为证据和改进闭环。

### 3.2 模型子系统目标

1. 用经典量化因子自动提名候选买卖点。
2. 通过人工二次审核与编辑形成高质量训练标签。
3. 训练出可解释、可迭代、可部署的买卖点预测模型。

## 4. 业务边界

### 4.1 系统 A：盲训（独立闭环）

输入：

1. 真实历史 K 线数据

输出：

1. 会话结果
2. 复盘报告
3. 个人决策能力提升

### 4.2 系统 B：模型训练（独立闭环）

输入：

1. 因子候选信号
2. 人工审核后的确认标签

输出：

1. 数据集版本
2. 模型版本
3. 盘中提醒能力（第二期）

### 4.3 共享能力

1. 行情数据接入与缓存
2. K 线展示组件
3. 标签与版本化存储
4. 任务调度与日志审计

## 5. 目标用户

1. 主用户：个人交易者（以提升自身决策与执行能力为核心）。
2. 扩展用户：希望用个人交易认知构建辅助模型的研究型用户。

## 6. 成功指标

### 6.1 盲训子系统 KPI

1. 周训练会话数
2. 单会话执行完成率
3. 胜率、盈亏比、回撤控制改善趋势
4. 复盘执行闭环率（训练后完成复盘占比）

### 6.2 模型子系统 KPI

1. 候选信号人工通过率
2. 标签一致性（重复审核一致率）
3. 模型 Precision/Recall/F1
4. 信号提醒采纳率与有效率

## 7. 分阶段业务价值

### Phase 1

1. 盲训能力可稳定使用
2. 模型数据生产链路打通（候选 -> 人审 -> 数据集）

### Phase 2

1. 训练出首个可用模型
2. 实现盘中买卖点提醒

### Phase 3

1. 人机协同闭环增强
2. 标签与模型持续迭代

## 8. 非目标

1. 不做自动下单交易。
2. 不做多用户社区与社交产品。
3. 不做高频撮合和超低延迟系统。
4. 不以“全自动量化策略替代人工”为阶段目标。
