# 盲训多会话资金同步问题修复记录

**日期：** 2026-04-26

## 问题现象

1. 盲训会话中，交易执行和结算结果本身是正确的。
2. 点击 `下一题` 后，新会话的初始资金偶发回到 `10w`。
3. 切回 `训练总览` 后，上方账户汇总区仍显示 `10w`，训练成果和账户日志没有同步。
4. 但页面下方的训练复盘明细已经能看到最新会话，说明“会话明细”与“账户聚合”出现了分叉。

## 根因

这是一个典型的多会话状态同步问题，涉及三层数据源：

1. 盲训工作台本地 state。
2. 主进程 SQLite 的 `training_sessions` / `training_profiles`。
3. 训练总览页对已完成会话的聚合统计。

核心问题有两个：

1. 会话切换时，新的会话起始资金仍可能读到旧的 `activeProfile.current_capital`，而不是当前会话刚刚结算出的最终资金。
2. 历史上存在一批 `training_sessions.status = 'active'` 的“挂起会话”，它们已经有动作和复盘数据，但从未被真正标记为 `finished`，导致总览上方只会统计到空结果。

## 修复

1. 盲训工作台在切题前优先使用当前会话的最终资本快照，而不是依赖 profile 刷新时序。
2. 从盲训页离开时，先显式结算当前会话，再刷新 `activeProfile`、`profileList` 和 `sessionList`。
3. 主进程 `db:finishSession` 改为幂等，并在完成后重建 profile 聚合，避免重复累计。
4. 在 `profile:list`、`profile:getActive`、`profile:getStats`、`db:listSessions` 读取前，自动修复可安全回补的挂起会话，补齐 `finished_at`、`final_capital` 和 `session_reviews`。

## 验证

1. 盲训结算后点击 `下一题`，新会话会继承上一题结算后的资金。
2. 切回 `训练总览` 时，账户汇总区会读取到最新 profile 资金。
3. 训练复盘明细继续正常展示历史会话，不受聚合修复影响。

## 经验

这类问题通常不是“某个 state 没 set”，而是“同一业务事实被多个层级各自维护”，一旦缺少统一结算点，就会出现：

1. 明细能看见，汇总看不见。
2. 新会话起始值和历史最终值不一致。
3. 旧 session 以 active 状态长期挂起，后续统计全部偏掉。

以后遇到类似问题，优先检查：

1. 数据是否有唯一真相源。
2. 切页 / 关页 / 连续训练是否共用同一结算入口。
3. 是否有历史脏数据需要一次性修复。
