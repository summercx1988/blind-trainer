## 一、主流量化交易系统架构调研

### 1.1 行业标准 Pipeline

一个完整的机构级量化交易系统通常包含 **七大核心模块**，形成从数据到执行的完整闭环：

```
数据接入 → Alpha研究/因子发现 → 信号生成 → 组合构建 → 风险管理 → 执行引擎 → 回测与评估
    ↑                                                                     │
    └────────────────────── 绩效监控 / 因子衰减检测 / 模型迭代 ←─────────────┘
```

**七大模块职责概要：**

| # | 模块 | 核心职责 | 关键指标/产物 |
|---|------|---------|-------------|
| 1 | 数据接入与管理 | 多源行情获取、清洗、存储、分发 | 时序DB、复权数据、数据质量门控 |
| 2 | Alpha 研究 | 因子定义、单因子检验（IC/IR/分箱）、因子入库 | RankIC > 0.03, ICIR > 0.5 |
| 3 | 信号生成 | 多因子合成（线性加权/树模型/深度学习） | 预测值、概率值、信号方向 |
| 4 | 组合构建 | 优化器（均值-方差/风险平价/HRP）、约束管理 | 持仓权重、换手率约束 |
| 5 | 风险管理 | VaR/CVaR、Barra归因、止损熔断、仓位管理 | 回撤限额、暴露控制 |
| 6 | 执行引擎 | OMS、执行算法（TWAP/VWAP/IS）、撮合模拟 | 成交滑点、执行成本 |
| 7 | 回测与评估 | 事件驱动回测、Walk-Forward、过拟合检测 | 夏普 > 1.0、最大回撤 < 20% |

### 1.2 主流开源框架对比

| 维度 | Zipline | Backtrader | LEAN (QuantConnect) | vn.py | NautilusTrader |
|------|---------|------------|---------------------|-------|----------------|
| **语言** | Python | Python | C#/Python | Python | Rust/Python |
| **架构** | Pipeline驱动 | 事件驱动 | 模型四层解耦 | 事件驱动+模块化 | 高性能事件驱动 |
| **市场** | 美股为主 | 全市场 | 全球多资产 | 中国市场为主 | 全球多资产 |
| **实盘** | 不支持 | IB/OANDA | 多券商 | CTP/50+接口 | 多交易所 |
| **社区** | 停更 | 活跃 | 非常活跃 | 中国最活跃 | 增长中 |
| **性能** | 中 | 低 | 高 | 中 | 极高 |
| **适用场景** | 学术研究 | 策略开发 | 机构级生产 | 中国实盘交易 | 高频/机构级 |

**核心架构模式总结：**

1. **LEAN 四模型架构**（业界典范）：Alpha Model → Risk Model → Portfolio Construction Model → Execution Model，四个模型完全解耦，可独立替换
2. **vn.py 三层分离**：Gateway（接口网关）→ EventEngine（事件引擎）→ App（应用层），50+ 券商接口统一封装
3. **Zipline Pipeline API**：因子计算与策略逻辑分离，Factor/Filter/Classifier 三种计算原语
4. **NautilusTrader**：Rust 核心 + Python 接口，研究-生产一致性设计

## 二、当前项目系统全景

### 2.1 项目定位

**幻化量方** — 面向个人交易者的人机协同交易训练与量化建模平台。核心定位：

- 不是全自动量化交易系统（不做自动下单）
- 不是高频交易系统（不做微秒级延迟）
- 是一个「数据 → 因子 → 标签 → 训练 → 推理 → 反馈」的 **ML 驱动买卖点预测平台**
- 同时包含一个独立的「盲训子系统」用于提升主观交易决策能力

### 3.2 架构差异分析

| 维度 | 行业标准 | 当前项目 | 评价 |

| **编程语言** | Python+C++/Rust | TypeScript+Python | ✅ 桌面端合理选择 |

### 3.3 差异化优势

与主流框架相比，当前项目有以下独特价值：

1. **人机协同定位**：不做全自动交易，而是训练人的决策能力（盲训）+ 辅助人的信号提醒（模型），这是市场空白
2. **标签质量闭环**：贪心上涨段 + 人工审核 + 可交易过滤 + 数据集版本化，比纯自动标签更可靠
3. **波段趋势分割**：ZigZag + PELT 双引擎设计，比简单 Swing Point 更有理论基础
4. **全链路桌面化**：从数据到推理到反馈，一个 Electron 应用闭环，无需部署服务
5. **反馈回流机制**：信号提醒 → 用户反馈 → 样本回灌 → 再训练，形成了数据飞轮雏形

## 八、总结

### 核心判断

1. **上半身（数据→因子→标签→训练→推理）已基本建成**，链路完整可运行，从数据到模型的生产管线在个人工具级别已经超过很多开源框架的完成度。

2. **下半身（因子验证→策略回测→风险归因→执行优化）是当前主要短板**，Alpha 研究引擎刚起步（40%），Walk-Forward 和经典策略 Benchmark 尚未实现，这是决定模型是否真正有效的关键。

3. **差异化定位清晰**：不做全自动量化系统，而是「人机协同」——盲训提升人的能力 + 模型辅助人的判断。这个定位在开源社区中是独特的。

4. **最大风险是模型有效性未经验证**：当前 AUC ~0.51（接近随机），样本量不足（~766），缺少 Walk-Forward 验证。需要尽快通过扩大数据量、改进标签质量、引入 Benchmark 对比来验证 ML 模型是否真正有预测力。
