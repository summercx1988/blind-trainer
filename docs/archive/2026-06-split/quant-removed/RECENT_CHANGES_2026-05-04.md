# 近期变更同步（截至 2026-05-04）

## 1. 策略打标与样本流转
- 重构波段标签生成与审查链路，强化 `buy-sell` 成对标注与多段呈现能力。
- 打标任务支持中止，避免全市场误参数导致长时间阻塞。
- 引入按轮次（run）覆盖与管理机制，支持 run 命名、按 run 抽样审查、按 run 冻结入库。
- 数据集管理从“单标签明细导向”调整为“run 导向全量管理”，补齐从抽样审查到数据集落地的衔接。

## 2. 数据与存储层
- 明确主行情库与业务库职责，标签结果独立存储并在展示层与主行情库关联读取。
- 补充市场库路径解析与配置能力（`marketDb`），用于主库 K 线读取与回退逻辑。
- 清理旧策略/旧打标相关代码与脚本，减少历史技术债（含部分废弃模块移除）。

## 3. 模型训练工作台调整
- 移除训练阶段“信号审核”页面（已转移/弱化），精简训练主流程：
  - 标签与数据集
  - 特征与训练
  - 集成实验
- 修复并强化标签审查 UI（K 线可视范围、成交量展示边界、配对状态反馈等）。

## 4. 模型部署与交互体验
- 修复“个股预测”白屏风险：对预测返回进行结构化校验与容错解析，异常返回不再导致渲染崩溃。
- 部署工作台新增“回测复盘”入口，接入完整回测报告能力（收益、回撤、阈值敏感性、模型对比）。
- 新增“推荐复盘”页（近半年默认视角）：
  - 支持模型/周期/时间段过滤
  - 支持 `T+N` 可配置评估
  - 输出推荐胜率、平均收益、平均回撤
  - 支持“仅看已评估”过滤
  - 增加按模型/按股票轻量汇总
  - 支持逐条推荐点击查看 K 线（直接读取主库行情，不新增存储）

## 5. 质量与工程侧
- 前后端 IPC、preload、类型声明同步更新，确保部署新增能力全链路可调用。
- 本轮改动已完成本地构建验证（`npm run build` 通过）。
