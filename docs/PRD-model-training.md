# PRD - 模型训练子系统

**版本：** v1.0  
**状态：** 执行版  
**日期：** 2026-04-05

## 1. 产品定位

模型训练子系统是独立于盲训的业务闭环，目标是“趋势/波段候选 + 人工审核 + 数据集冻结 + 训练评估”。模型部署、预测、提醒反馈和再训练由独立的模型部署工作台承接。

## 2. 核心用户故事

1. 作为交易者，我希望系统先用经典量化因子提名潜在买卖点。
2. 作为交易者，我希望对候选点逐条审核、编辑、确认，确保符合我的交易认知。
3. 作为交易者，我希望只用已确认样本训练模型，并清楚知道每个模型来自哪版数据。
4. 作为交易者，我希望训练完成后在模型部署页激活模型、查看买卖点提醒，并记录反馈。

## 3. 功能范围

### 3.1 P0 功能

1. 因子候选生成：
   - 经典因子策略库（MA、RSI、MACD、BOLL、量价等）
   - 输出候选买卖点与置信度
2. 人工审核工作台：
   - 候选列表
   - 审核动作：`接受 / 拒绝 / 编辑`
   - 标签动作审计日志
3. 数据集版本化：
   - 草稿 -> 冻结
   - 冻结后只读
4. 模型训练任务：
   - 选择数据集版本 + 特征规格
   - 输出模型版本与评估结果
5. 模型评估页：
   - 分类指标
   - 时间切分表现
   - 样本覆盖与误差分析

### 3.2 P1 功能

1. 特征规格版本管理
2. 候选策略效果对比
3. 主动学习抽检（优先人工审核不确定样本）

### 3.3 P2 功能（第二期）

1. 盘中实时提醒：
   - 最新 bar 推理
   - 信号提醒与置信度
2. 提醒反馈回流：
   - 用户采纳/忽略
   - 反馈进入后续训练样本

## 4. 非功能要求

1. 所有训练任务必须可追溯（数据版本、特征版本、参数、时间）。
2. 所有模型结果必须可复现。
3. 人工审核操作必须有审计记录。

## 5. 数据要求

最小实体：

1. `signal_candidates`
2. `candidate_review_logs`
3. `dataset_versions`
4. `dataset_items`
5. `feature_specs`
6. `model_versions`
7. `model_evaluations`
8. `signal_events`（第二期）
9. `signal_feedback`（第二期）

关键原则：

1. 只有人工确认标签可进入训练集。
2. 模型训练与评估必须按时间切分，避免未来泄漏。
3. 每个提醒信号必须可追溯到模型版本。

## 6. 推荐算法路线

### 6.1 第一阶段

1. 候选信号：规则与因子组合
2. 监督学习：LightGBM/XGBoost 基线

### 6.2 第二阶段

1. 逐步引入更复杂序列模型
2. 不替代人工审核机制

## 7. 当前代码映射

已存在基础能力：

1. 标签表与标签写入能力
2. 基础模型页面入口与 IPC 框架
3. Python 目录与特征构建脚手架
4. 候选信号表与审核日志表已落库，已支持 MA20 上下穿候选生成 IPC（v1）
5. 人工审核台前端已接入，可执行“接受/拒绝/编辑”并写入审计日志
6. 候选因子已扩展为 MA/RSI/MACD 三类基础策略（v1）
7. 数据集版本化已具备最小闭环：可从已接受候选创建草稿，并执行冻结
8. 已接入特征构建任务入口：可从冻结数据集触发 `feature build` 并记录任务状态
9. 已接入模型训练任务入口：可消费冻结数据集+特征规格，产出模型版本与评估记录（Baseline v1）
10. 模型版本管理可用：支持模型列表查询与激活切换
11. 评估管理增强：支持评估明细查询与双模型指标对比（前端）
12. 盘中提醒入口（W9）已接入：可按标的与周期触发最新 bar 推理，并按阈值写入 `signal_events`
13. 提醒反馈闭环（W10 起步）已接入：可对提醒执行 `accept/ignore/modify`，写入 `signal_feedback` 与事件状态
14. 反馈回流最小链路已接入：`accept/modify` 会同步生成 `accepted` 候选，下一次数据集草稿可直接纳入（并按 code+period+bar+signal 去重）
15. 自动提醒扫描已接入：在 `data:sync` 后自动按最新行情标的批量推理，并对同模型同bar同方向提醒去重
16. 再训练入口已接入：支持反馈批量回放、手动触发再训练，并记录 `retraining_runs`（可选训练成功后自动激活）
17. 增量再训练入口已接入：默认基于上次成功 run 的 `finished_at` 作为回放起点（`sinceCreatedAt`）
18. 特征构建已升级为真实数据优先：`feature build` 优先读取 SQLite 数据集样本并按样本时间点计算特征；数据集不可用时保留 mock 回退
19. 工程治理增强：数据库已接入 `user_version` 迁移机制，并增加提醒事件去重唯一索引迁移
20. 严格真实数据模式已接入产品界面：模型训练工作台可切换 strict/fallback，并在 strict 缺样本时返回可读失败原因
21. 候选引擎因子覆盖已扩展到 MA/RSI/MACD/BOLL/量价突破，审核台因子展示已同步中文标签
22. 数据集版本治理增强：已支持冻结版本对比（重叠率/差异样本预览）与回滚创建新草稿
23. 数据集草稿新增冲突样本策略开关：`keep_all`（保留全部标签）/`single_best`（同bar保留最高分）
24. 数据集治理新增“策略效果评估”与“跨版本自动合并”入口：可评估冲突bar规模，并按策略将两版冻结数据集合并为新草稿
25. 冲突治理已具备自动推荐与留痕：系统会基于冲突规模推荐策略，并记录推荐/选择/产出结果用于回看
26. 已增加策略趋势分析与自动调参建议：基于历史留痕统计一致率、冲突均值与策略效果，给出草稿/合并默认策略建议
27. 已支持按标的/周期分层趋势分析：可查看分组一致率、冲突均值、样本规模与分组建议策略
28. 已支持“策略-训练结果联动分析”：可按策略/标的/周期查看模型 Test 指标、提醒反馈采纳率与建议策略
29. 已支持“策略-PnL归因（盲训联动）”：可按策略/标的/周期查看盲训会话收益率、盈利会话率与回撤表现
30. 已支持“提醒-交易闭环归因”初版：可按反馈动作/策略/标的/周期查看提醒事件到盲训会话的覆盖率与收益表现
31. 标签默认口径已切换为 `exit_return`（实际退出收益），并保留 `max_return` 兼容开关用于历史对照
32. 数据集管理新增“检查标签”入口与“标签人工抽样检查”面板，可并排查看 K 线、实际收益、窗口最高收益和 `label_alignment`
33. 特征切分策略已从 `70/15/15` 调整为 `70/10/20`，扩大 test 集以增强 OOS 与回测校验

当前缺口：

1. 候选审核台的“编辑后落标签细化（置信度/备注结构化）”仍需加强
2. 数据集版本化仍缺“提醒事件 -> 真实下单成交结果”生产级闭环归因（当前已覆盖策略层 + 模型评估 + 提醒反馈 + 盲训PnL联动 + 盲训会话匹配闭环）
3. 模型训练仍为轻量基线，缺少 LightGBM/XGBoost 与更稳健的时间切分评估增强
4. 模型评估仍缺阈值/分层/稳定性分析与自动报告模板
5. 自动提醒当前挂在 `data:sync` 触发，尚未接入独立行情订阅服务（准实时）
6. 再训练仍缺定时调度、失败重试和运行资源配额控制
7. 真实特征链路仍需“禁用 mock 回退”的生产级默认策略与失败告警策略

## 8. 验收标准

1. 能从行情数据生成候选点并进入人工审核台。
2. 能冻结数据集版本并触发训练任务。
3. 能产出可查询的模型版本与评估报告。
4. 第二期可在盘中输出可追溯提醒信号。
