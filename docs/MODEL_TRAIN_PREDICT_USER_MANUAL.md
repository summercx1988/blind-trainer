# 模型训练与预测用户手册

> 适用版本：当前 `Model Pipeline + Model Deployment` 双工作台  
> 更新时间：2026-04-27  
> 目标读者：产品、交易研究、标注审核、模型运营

---

## 1. 你会完成什么

按本手册操作后，你可以完成一条完整闭环：

1. 用真实行情生成趋势/波段候选信号并人工审核。
2. 冻结训练数据集，构建特征，训练买点/卖点模型。
3. 选择活跃模型并进行准实时预测。
4. 收集提醒反馈并触发再训练，持续迭代模型。

---

## 2. 页面分工（先理解，再操作）

### 2.1 数据管理（Data Ops）
- 负责行情底座：股票池初始化、K 线增量同步、覆盖检查。
- **不负责**训练样本标注与冻结。

### 2.2 模型训练（Model Pipeline）
- 负责离线产模链路：
  - 候选审核
  - 数据集冻结
  - 特征工程
  - 训练与评估
  - 集成实验

### 2.3 模型部署（Model Deployment）
- 负责在线运营链路：
  - 模型仓库
  - 准实时预测
  - 提醒反馈
  - 再训练

## 3. 标准工作流（用户操作版）

## 3.1 第 0 步：准备数据（Data Ops）

### 操作
1. 进入「数据管理」页面。
2. 首次使用先执行「初始化数据」。
3. 日常执行「增量同步」：
   - 同步数量：建议按当天运营节奏配置。
   - 周期：至少包含你后续预测要用的周期（`1d` / `15m` / `5m`）。
4. 在「股票列表」确认覆盖与最近同步时间。

### 验收
- 股票池非空。
- 目标周期有覆盖（特别是你要训练/预测的周期）。

---

## 3.2 第 1 步：候选审核（模型训练 > 候选审核）

### 目的
把“技术触发点”过滤成“结果导向的可盈利样本候选”。

### 操作
1. 选择 `标的代码`、`周期`、`回看条数`，点击「生成候选」。
2. 可在「结果门槛参数」里按周期调整门槛（会持久化保存）：
   - 前瞻窗口、最小未来 bars
   - 买点：最小浮盈/退出收益、最大回撤、最小 RR
   - 卖点：最小下跌、最大反弹、最小 RR
3. 在「人工审核台」逐条执行：
   - 通过（accept）
   - 拒绝（reject）
   - 编辑（edit，填写说明）

### 如何看字段
- `前瞻(退出/浮盈/回撤)`：候选点后续真实结果概览。
- `RR`：风险收益比，越高越好（需结合样本稳定性看）。
- `趋势` + `结果标签`：确认是否符合趋势/波段策略预期。

### 验收
- 已接受样本中，买卖点比例大致符合你的策略预期。
- 拒绝理由集中且可解释（不是随机噪声拒绝）。

---

## 3.3 第 2 步：数据集冻结（模型训练 > 数据集冻结）

### 目的
把已审核样本固化为可复用训练集。

### 操作
1. 设置草稿参数：
   - 草稿名称（可选）
   - 代码/周期过滤（可选）
   - **样本来源过滤**：
     - `仅门槛通过样本（默认）`（推荐）
     - `全部已接受样本`
   - 冲突策略：
     - `保留全部标签`
     - `同bar仅保留最高分`
2. 点击「评估冲突策略效果」查看冲突统计与建议。
3. 点击「创建草稿」。
4. 核查样本数后点击「冻结」。

### 验收
- 数据集状态为 `frozen`。
- 样本量满足训练最低要求（建议买点/卖点各自都达到可训练规模）。

---

## 3.4 第 3 步：特征工程（模型训练 > 特征工程）

### 操作
1. 选择冻结数据集。
2. 选择特征规格版本（例如 `v010`）。
3. 视场景选择「严格真实数据模式」：
   - 生产前验证建议开启。
4. 点击「触发构建」。

### 验收
- 任务状态为 `succeeded`。
- 无特征构建错误信息。

---

## 3.5 第 4 步：训练与评估（模型训练 > 训练 & 评估）

### 操作
1. 选择冻结数据集、特征规格、任务类型、引擎、trials。
2. 任务类型建议分开训练：
   - `buy_signal`（买点模型）
   - `sell_signal`（卖点模型）
3. 启动训练并等待完成。
4. 在「模型对比」查看指标并挑选候选模型。
5. 需要上线时，点击「激活」。

### 关键建议
- 引擎默认推荐 `CatBoost`（页面提示）。
- Trials 不够会导致调参不足，过高会延长训练时间。
- 不要只看单一指标，至少一起看 `AUC/F1/Precision/Recall`。

---

## 3.6 第 5 步：模型仓库运营（模型部署 > 模型仓库）

### 操作
1. 若磁盘已有离线模型文件，先点「扫描并导入」同步产物。
2. 查看活跃模型状态、推荐候选、最近信号。
3. 对模型做激活、重命名、备注、报告查看。

### 验收
- 有明确 `active` 模型。
- 活跃模型指标与当前策略阶段一致。

---

## 3.7 第 6 步：准实时预测（模型部署 > 准实时预测）

### 操作
1. 选择模型（或默认使用活跃模型）。
2. 输入单标的或多个代码（逗号分隔）。
3. 选择周期后点击「预测」。
4. 可配置并保存：
   - 自动刷新开关
   - 刷新间隔（秒）
   - 各周期“数据新鲜度阈值（分钟）”

### 结果解释
- `信号`：买点/卖点方向。
- `分数` + `置信度`：排序和过滤参考。
- `延迟` + `超阈值标的`：判断当前预测是否可执行。

---

## 3.8 第 7 步：提醒反馈与再训练（模型部署 > 提醒反馈 / 再训练）

### 提醒反馈
1. 对每条提醒执行：
   - 采纳（accept）
   - 修正（modify）
   - 忽略（ignore）
2. 必要时手动点「手动回填提醒」，把反馈同步到候选池。

### 再训练
1. 在「再训练」选择规格、任务类型、样本上限。
2. 触发：
   - 全量再训练
   - 增量再训练
3. 查看运行记录、摘要、是否已激活。

## 4. 回测与年化收益（上线前建议）

当前系统支持 CLI 回测（用于查看 `annualized_return` 等指标）：

```bash
PYTHONPATH=python python3 -m trading_trainer.cli model backtest \
  --model <model_id> \
  --threshold 0.5 \
  --holding-days 2 \
  --strategy-type model \
  --exit-mode fixed_holding \
  --db "~/Library/Application Support/stock-trading-simulator/stock-trading.db"
```

阈值优化：

```bash
PYTHONPATH=python python3 -m trading_trainer.cli model optimize-threshold \
  --model <model_id> \
  --objective sharpe \
  --holding-days 2 \
  --db "~/Library/Application Support/stock-trading-simulator/stock-trading.db"
```

> 注意：若回测有效交易天数很短，年化会显著失真。请结合累计收益、最大回撤、交易天数和交易笔数一起判断。

## 5. 常见问题（FAQ）

### Q1：为什么“创建草稿”后样本很少？
- 通常是因为：
  - 样本来源过滤使用了 `qualified_only`（这是预期）
  - 候选审核通过数量不足
  - 代码/周期过滤过严

### Q2：为什么预测结果提示数据延迟高？
- 本地行情未及时同步。先回到「数据管理」做增量同步，再预测。

### Q3：买点和卖点要不要分开训练？
- 建议分开（`buy_signal`、`sell_signal` 两模型），便于分别调参与风控。

### Q4：模型指标不错但实盘信号一般？
- 先检查：
  - 训练样本是否过于“短窗”
  - 阈值是否过低（可用 `optimize-threshold`）
  - 新鲜度阈值和数据同步是否稳定
  - 反馈闭环是否持续运行

## 6. 运营建议（简版）

1. 每天：先同步数据，再跑预测，再做反馈。  
2. 每周：复查候选门槛参数与样本质量。  
3. 每双周：做一次再训练 + 回测复核。  
4. 任何时候：避免只看单个高年化数字，必须看回撤和样本稳定性。
