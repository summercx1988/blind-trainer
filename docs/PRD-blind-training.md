# PRD - 盲训子系统

**版本：** v1.0  
**状态：** 执行版  
**日期：** 2026-04-05

## 1. 产品定位

盲训子系统是一个独立产品闭环，目标是提升用户主观交易决策能力，不依赖模型子系统上线。

## 2. 核心用户故事

1. 作为交易者，我希望在真实历史 K 线上随机起点训练，避免看见未来。
2. 作为交易者，我希望按真实可执行规则做买卖持有决策。
3. 作为交易者，我希望每次训练后能看到结构化复盘结果，知道错在哪。

## 3. 功能范围

### 3.1 P0 功能

1. 真实样本抽取：
   - 从本地历史 K 线样本池随机选取
   - 支持 `5m / 15m / 1d`
2. 盲训执行：
   - 仅显示当前及历史 bar
   - 动作：`买入 / 卖出 / 持有 / 结束`
   - 规则：仅做多、bar close 成交、手续费、T+1
3. 会话持久化：
   - 保存 `training_session`
   - 保存 `trade_actions`
4. 基础复盘：
   - 收益与胜率
   - 交易次数与持仓表现
   - 关键操作时间点回放
5. 训练记录页：
   - 展示真实会话历史
   - 支持按时间和周期筛选

### 3.2 P1 功能

1. 决策质量评分：
   - 入场效率
   - 出场效率
   - 纪律分
2. 跳过样本质量评估
3. 多会话统计面板

## 4. 非功能要求

1. 主操作响应快，单步推进不卡顿。
2. 会话中断可恢复。
3. 数据写入可靠，异常时有可读错误提示。

## 5. 数据要求

最小实体：

1. `training_sessions`
2. `trade_actions`
3. `session_reviews`（新增，复盘结果）

关键原则：

1. 训练结果必须可回溯到样本、周期、规则配置。
2. 复盘指标必须可复算，避免纯展示字段。

## 6. 交互要求

1. 页面主层级清晰：样本信息、账户状态、动作区、K 线区、日志区。
2. 动作反馈即时可见，避免“点了没反应”。
3. 关键风险状态醒目：不可买时间窗、T+1 不可卖、资金不足。

## 7. 当前代码映射

已存在基础能力：

1. 盲训页面与买卖逻辑
2. 会话与动作数据库落库
3. 基础行情同步能力

当前缺口：

1. 真实样本抽取已接入数据库，但在样本不足时仍会回退到 mock（需进一步提升真实样本覆盖率）
2. 训练记录页已切换到真实会话与动作明细，并支持时间/周期/状态筛选；尚缺更细粒度统计报表
3. 复盘评分已具备基础可复算指标（胜率、收益率、回撤、平均持仓 bar），高级评分模型仍待迭代

## 8. 验收标准

1. 训练全流程不依赖前端随机样本可独立完成。
2. 训练记录页全部来自数据库真实会话。
3. 同一会话可复现关键指标结果（收益、胜率、交易次数）。
4. 构建通过，主链路无阻断性错误。
