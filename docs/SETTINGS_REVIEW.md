# 盲训工作台设置面板 — 问题诊断与改造方案

## 一、问题清单

### P0 — 功能性 Bug

| # | 问题 | 根因 | 影响 |
|---|------|------|------|
| 1 | **应用配置后不刷新** | `onApplySettings` 在 idle 状态只设 `setInitialized(false)`，未触发样本重新加载流程 **（修订：根因需先 grep 确认——若存在 `useEffect(()=>loadSamples(), [initialized])`，则问题在依赖不全或 effect 早退，而非未触发）** | 改了样本池深度/抽样范围，样本池不变 |

### P1 — UX 设计问题

| # | 问题 | 分析 |
|---|------|------|
| 4 | **周期设置只有一个选项** | 当前只有 `1d`（日线），设置面板中保留一个单选按钮纯粹是冗余占位 |
| 5 | **"补载更多K线"按钮定位不清** | 该功能调用 `extendActiveSample()`，重新从数据库拉取该股票全部 K 线以扩展当前样本长度。但：① 训练到样本末尾时代码已自动调用（[L502](../src/components/trading/BlindTrainingWorkbench.tsx#L502)）；② 手动点击可能在训练中导致 currentBarIndex 映射错乱；③ 用户不知道什么时候该点 |
| 6 | **成交模式名称不直观** | "盘尾收盘"/"次根开盘" — 用户难以理解实际区别（当前K线收盘价成交 vs 下一根K线开盘价成交） |

| 8 | **最低股价位置尴尬** | 在样本池/抽样范围设置中混入了一个数字输入框，视觉割裂 |

## 三、改造方案

### Phase 1：修复即时 Bug

> **修订：Phase 1 的所有修复动作前应先做根因验证**，不要直接按文档描述动手。详见各小节。

#### 1.1 应用配置后立即刷新样本池

**当前行为**：idle 状态下 `onApplySettings` 只设 `setInitialized(false)`，但没有触发样本重新加载。

**修复**：idle 时应用配置后直接触发样本重新加载流程（与初始化时的流程一致）。

```
onApplySettings → 设置更新 → setInitialized(false) → 触发 loadSamples → 样本池刷新
```

**涉及文件**：`BlindTrainingWorkbench.tsx`

**（修订）根因验证步骤**：
1. `Grep` 搜索 `useEffect` 中是否引用 `initialized` / `setInitialized` 触发 `loadSamples`
2. 若存在，根因是依赖数组不全 / 条件分支提前 return，应修 effect 而非在 onApplySettings 里加手动 reload
3. 若不存在，按文档方案修

#### 1.2 K线 +- 按钮生效

**当前行为**：`BaseKlineChart` 使用 `chart.setDataLoader()` 注册数据加载器，但 klinecharts 在 loader 注册后不会主动重新获取数据。`visibleCount` 变化后 `visibleBars` 数组引用变了，但图表内容不刷新。

**修复**：改用 `chart.applyNewData(data)` 直接设置数据。`visibleCount` 变化时图表立即更新可见范围。

**涉及文件**：`BaseKlineChart.tsx`

**关键改动**：
```typescript
// 之前（不触发刷新）
chart.setDataLoader({ getBars: handleGetBars })

// 之后（直接设置数据，立即刷新）
const klineData = data.map(bar => ({
  timestamp: toMs(bar.timestamp),
  open: bar.open, high: bar.high, low: bar.low, close: bar.close, volume: bar.volume
}))
chart.applyNewData(klineData)
```

**（修订）API 与性能考量**：
- `applyNewData` 会**丢失** klinecharts 的增量/滚动加载能力，1560 根数据下需验证是否仍有滚动到末尾补载的需求
- 应核对当前 klinecharts 版本是否提供 `updateData()` / 重新触发 loader 等更轻量的方案
- 1560 根 × 多技术指标的渲染性能需要在 1.2 修复完成后**实测一遍**（M1 Mac 最低配为基准）

### Phase 2：精简设置面板

#### 2.1 移除冗余项

| 移除项 | 原因 |
|--------|------|
| 周期选择器（仅 `1d`） | 只有一个选项，纯冗余 |
| 补载更多K线按钮 | 已有自动扩展逻辑（`extendActiveSample` 在训练到末尾时自动调用） |

**（修订）补充**：移除"补载更多K线"按钮后，应在工作台/样本池附近加一行**轻量灰字提示**（如"样本不足时自动扩展"），否则用户会以为"我没操作就没了"。

#### 2.2 样本池深度（从设置面板移除）

```
之前：260 / 520 / 1040 / 1560（4个数字选项，在设置面板内）
之后：从设置面板移除，改由 K 线图下方的数字控件（+/- 步进）控制
```

**（修订）单一控制源**：样本深度的调整面与 K 线的可见条数本质上**是同一个量**——你能在 K 线上看到多少根，就能在训练中过多少根。把它从设置面板拿掉，避免用户面对"两个相关参数"产生认知冲突。**P0 #2（修复 +/- 按钮）必须先于本变更落地**，否则用户会失去唯一的深度控制。

**遗留问题（需后续设计确认）**：
- 控件形态：**滑块** ✅
- 步长：滑块连续，**每次跳 1 根** ✅
- 系统默认：**120 根（约 0.5 年日线）** ✅
- 训练中调整 visibleCount：本质是"热生效"（仅调整可见窗口大小，不需要重置 session） ✅

#### 2.3 抽样范围（替换为 4 档新值）

```
之前：40只 / 80只 / 200只 / 500只（4个选项）
之后：200只 / 500只 / 1000只 / 2000只（4个选项）
```

**（修订）整体上移、放弃小样本档**：40/80 这种"快速过题"档位使用率低，**直接被 1000/2000 替换**，整体重心从"小规模过题"上移到"中大规模实战训练"。仍保留 4 档数量，便于直接对比选择。

### Phase 3：改善交互细节

#### 3.1 成交模式重命名

```
之前：盘尾收盘 / 次根开盘
之后：当天收盘价 / 次日开盘价
```

加 tooltip 说明：
- 当天收盘价："以**当前K线**的收盘价成交（t 时刻），适合模拟尾盘买入/卖出"
- 次日开盘价："以**下一根K线**的开盘价成交（t+1 时刻），更接近真实交易"

**（修订）tooltip 补充**：除了语义，必须明确**时序**（是当前 K 收盘还是下一根 K 开盘），否则"次日"二字仍有歧义（可能被理解为 T+1 任意时刻）。明确写"下一根K线"。

#### 3.2 最低股价移到工具栏

将最低股价从设置面板移到走势筛选旁边，作为快速筛选条件：

```
[走势筛选: 全部 上涨 下跌 震荡]  [最低价: ___元]  [连续训练 ☐]
```

**（修订）布局风险**：工具栏已塞 4 个元素，横向空间会变紧——需要看实际宽度，可能要分两行或把"应用配置"改为图标按钮。

#### 3.3 应用配置按钮改进

**（修订）必须先区分设置项的"破坏性"**：

| 设置类别 | 示例 | 训练中点击应用 | 需要的确认 |
|---------|------|---------------|-----------|
| **会重置样本池的** | 走势筛选、抽样范围、最低股价 | 结束当前 session、重载 | ✅ 需要确认弹窗 |
| **热生效、不影响样本的** | 成交模式、连续训练模式、**K 线 +/- 调整可见条数** | 立即生效 | ❌ 不需要弹窗 |

**修订方案**：
- 对"破坏性"设置项：弹确认框 "应用新配置将结束当前训练，是否继续？"，确认后重载
- 对"热生效"设置项：直接保存 + 即刻应用，**不弹框**（避免无谓摩擦）
- 在 UI 上用视觉分组（分组标题或分割线）让用户能感知到"哪些是危险操作"
